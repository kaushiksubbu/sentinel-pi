### 4. Docker Foundation — Complete Conceptual Understanding
All concepts understood from first principles:

```
Why Docker:
→ Protects against host OS library corruption
→ March venv incident would not have occurred
→ Container carries own Python + libraries

Image vs Container:
→ Dockerfile = recipe (you write)
→ docker build = cooking (Docker executes)
→ Image = frozen packaged meal
→ Container = running instance

Encapsulation analogy:
→ Same OOP principle, larger scope
→ Image = class definition
→ Container = object instance

Volumes:
→ Data lives OUTSIDE container on host
→ Left = real Pi path, Right = container alias
→ Container deleted → data survives

Mount vs Bake:
→ Development: mount src/ — no rebuild on change
→ Production: bake src/ — versioned, immutable
→ CI/CD enforces code quality before bake

Ephemeral containers:
→ Run → complete → dissolve
→ One container in RAM at a time
→ Critical for Pi 8GB constraint
→ Same principle as serverless functions
→ Zero zombie processes

Docker Compose:
→ Defines 4 services + shared volumes + network
→ depends_on = startup order
→ Prefect = completion sequencing

sentinel-net bridge:
→ Private network for containers
→ Containers address each other by name
→ Isolated from host network
```
