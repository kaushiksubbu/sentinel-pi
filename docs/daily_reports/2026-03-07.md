## What Was Built Today

### `src/contracts.py`
Centralised data contract definitions for all sources. Defines known field aliases per source to handle schema drift without modifying transformation logic.

```python
ZIGBEE_CONTRACT = {
    "temperature": ["temperature", "temp", "tmp"],
    "humidity":    ["humidity", "hum", "rh"],
    "battery":     ["battery", "bat"],
}

KNMI_CONTRACT = {
    "temperature": ["ta"],
    "humidity":    ["rh"],
    "wind_speed":  ["ff"],
}
```

### `src/transform_zigbee_to_silver.py`
Bronze → Silver transformation for Zigbee data. Three independent modules:

- `read_zigbee_bronze()` — reads unprocessed rows, excludes Bath at query level
- `validate_zigbee_rows()` — parses JSON payload via contract, validates temp and humidity
- `write_zigbee_silver()` — governed write to weather_silver, updates watermark on success only
- `transform_zigbee_to_silver()` — thin orchestrator

### Validation Rules (Locked)

| Field | Valid Range | DQ Flag |
|-------|-------------|---------|
| indoor temp | 14 to 30°C | `temp_null` / `temp_out_of_range` |
| indoor humidity | 25 to 85% | `rh_null` / `rh_out_of_range` |
| Bath room | Excluded | Filtered at read stage |

### `ops/ops.db — watermarks`
Watermark table tracks last processed timestamp per source. Prevents duplicate processing. Updated only after successful Silver write — full rollback on failure.

```
source   last_processed              updated_at
knmi     2026-03-07 22:40:00         2026-03-07 23:32:xx
zigbee   2026-03-07 23:32:25         2026-03-07 23:33:xx
```

## Pipeline Flow (Complete)

```

Zigbee MQTT → landing_zone → zigbee_raw (Bronze) → weather_silver (Silver)
```

Both sources automated via cron. Both flowing into single canonical Silver table.
