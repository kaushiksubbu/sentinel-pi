### Percentage-Based DQ — Implemented
Replaced hardcoded record count thresholds with runtime completeness calculation:

```python
knmi_completeness   = knmi_valid_count / knmi_total_count
zigbee_completeness = zigbee_valid_count / zigbee_total_count
```

Config thresholds:
```python
KNMI_COMPLETENESS_MIN   = 0.84   # 84% minimum
ZIGBEE_COMPLETENESS_MIN = 0.94   # 94% minimum
```

**Why percentage over record count:**
- Streaming-ready — no hardcoded expected counts
- Self-calibrating — adapts to any collection frequency
- Eliminates manual threshold updates on pipeline changes

### Read-Only Connections — `db_utils.py`
Added `connect_to_db_readonly()` for all read operations:
```python
# AI scripts, manual queries, reporting
con = connect_to_db_readonly(GOLD_DB)
```
Prevents lock conflicts during 10-minute pipeline runs.

### Complete Hour Filter — Gold
Gold only aggregates Silver rows belonging to completed hours:
```sql
AND DATE_TRUNC('hour', observed_at) + INTERVAL '1 hour' < NOW()
```
Prevents partial hour aggregation — every Gold row is complete.
